# Практична робота: Побудова поведінкових UML-діаграм
**Предметна область:** Система ситуаційної обізнаності «Delta»
**Виконав:** Курсант Шмиголь Олександра

---

## 1. Діаграма варіантів використання (Use Case Diagram)

```mermaid
graph LR
    subgraph "Система Delta"
        UC1(Внести дані про ціль)
        UC2(Переглянути мапу)
        UC3(Верифікувати дані ML)
        UC4(Прийняти рішення)
    end

    Op[Оператор БПЛА] --- UC1
    Op --- UC2
    An[Аналітик] --- UC2
    An --- UC3
    Com[Командир] --- UC2
    Com --- UC4

    UC3 -.->|extend| UC1
```

---

## 2. Діаграма послідовності (Sequence Diagram)

```mermaid
sequenceDiagram
    autonumber
    participant Op as Оператор
    participant DB as База даних Delta
    participant ML as ML-Модуль
    participant An as Аналітик

    Op->>DB: Внесення координат цілі
    DB->>ML: Запит на перевірку достовірності
    Note right of ML: Reliability Check
    ML-->>DB: Ймовірність (%)
    DB->>An: Сповіщення про нову ціль
    An->>DB: Підтвердження/Відхилення даних
```
---

## 3. Діаграма діяльності (Activity Diagram)

```mermaid
graph TD
    A[Початок] --> B[Отримання даних від БПЛА]
    B --> C{Валідація формату}
    
    C -- Помилка --> D[Відхилити та повідомити]
    C -- Коректно --> E[Аналіз ML-алгоритмом]
    
    E --> F{Надійність понад 80?}
    
    F -- Так --> G[Візуалізація на мапі]
    F -- Ні --> H[Додаткова перевірка аналітиком]
    
    G --> I[Прийняття рішення командиром]
    H --> I
    
    D --> J[Кінець]
    I --> J
```
